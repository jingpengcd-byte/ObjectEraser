# LaMa图像修复演示系统使用教程

## 1. 系统概述

LaMa演示系统是一个基于Web的图像修复应用程序，集成了LaMa（Large Mask Inpainting with Fourier Convolutions）深度学习模型。用户可以通过简单的网页界面完成图像上传、掩码绘制、图像修复和结果下载的完整流程。

### 主要功能
- **图像上传**：支持PNG、JPG、BMP等多种格式
- **交互式掩码绘制**：提供画笔、矩形、橡皮擦等工具
- **高质量图像修复**：使用LaMa深度学习模型
- **实时预览**：修复过程实时显示
- **结果下载**：一键下载修复后的图像

### 技术栈
- **后端**: Python Flask
- **深度学习**: PyTorch + LaMa模型
- **前端**: HTML5 + CSS3 + JavaScript
- **画布操作**: Canvas API

## 2. 快速开始

- Ubuntu 18.04+ / CentOS 7+ / macOS 10.15+

## 3. 详细使用步骤

### 3.1 步骤1：上传图像

1. 打开浏览器，访问演示系统地址
2. 点击"选择文件"按钮或直接将图像拖放到上传区域
3. 支持的文件格式：PNG、JPG、JPEG、BMP、TIFF、WEBP
4. 图像文件大小限制：最大50MB

**注意事项：**
- 上传后可以在右侧看到图像预览
- 系统会显示图像的文件名、尺寸和分辨率
- 确保图像清晰，修复效果更佳

### 3.2 步骤2：创建掩码

1. 选择合适的绘图工具：
   - **画笔工具**：自由绘制需要修复的区域
   - **矩形工具**：绘制矩形区域
   - **橡皮擦工具**：擦除已绘制的区域

2. 调整画笔大小（5-100像素）
3. 在图像上标记需要修复的区域：
   - **白色区域**：表示需要修复的部分
   - **黑色区域**：表示保留原始内容

**操作技巧：**
- 使用滚轮可以缩放视图（如支持）
- 按住空格键可以拖动画布
- 使用`Ctrl+Z`或撤销按钮可以撤销操作
- 点击清除按钮可以重置整个掩码

### 3.3 步骤3：图像修复

1. 检查原始图像和掩码预览
2. 选择修复选项：
   - **标准修复**：快速修复，适合一般场景
   - **高质量修复**：启用精炼算法，效果更好但耗时更长

3. 点击"开始修复"按钮
4. 等待处理完成，实时查看修复进度

**处理时间参考：**
- 标准修复：约10-30秒（取决于图像大小）
- 高质量修复：约30-90秒（取决于图像大小）

### 3.4 步骤4：结果下载

1. 修复完成后，可以查看修复前后的对比
2. 下载修复结果：
   - 点击"下载结果"按钮
   - 文件将自动保存到本地
   - 默认文件名：`result_原始文件名.png`

3. 其他操作：
   - 点击"重新开始"进行新的修复
   - 点击"复制链接"获取结果的直接下载链接

## 4. 高级功能

### 4.1 批处理模式

如果需要批量处理多个图像，可以使用命令行工具：

```bash
# 进入LaMa项目目录
cd /home/guojingpeng/workSpace/lama

# 设置环境变量
export TORCH_HOME=$(pwd)
export PYTHONPATH=$(pwd)

# 批量修复
python3 bin/predict.py \
    model.path=$(pwd)/big-lama \
    indir=/path/to/input/images \
    outdir=/path/to/output/results
```

### 4.2 API接口

演示系统提供RESTful API接口，可以用于集成到其他系统：

```bash
# 1. 上传图像
curl -X POST -F "file=@/path/to/image.jpg" \
    http://localhost:5000/api/upload

# 2. 创建掩码（发送掩码数据）
curl -X POST -H "Content-Type: application/json" \
    -d '{"type": "points", "points": [{"x":100,"y":100}]}' \
    http://localhost:5000/api/create_mask

# 3. 执行修复
curl -X POST -H "Content-Type: application/json" \
    -d '{"refine": true}' \
    http://localhost:5000/api/inpaint

# 4. 下载结果
curl -X GET \
    http://localhost:5000/api/download/result_filename.png
```

## 5. 故障排除

### 5.1 常见问题

#### Q1: 上传图像失败
- **原因1**: 文件格式不支持
  - 解决方案：确保上传PNG、JPG等常见格式
- **原因2**: 文件大小超过限制
  - 解决方案：压缩图像或减小尺寸
- **原因3**: 网络连接问题
  - 解决方案：检查网络连接，重试上传

#### Q2: 修复结果不理想
- **原因1**: 掩码区域过大
  - 解决方案：缩小修复区域，分多次修复
- **原因2**: 图像质量较差
  - 解决方案：使用更清晰的原始图像
- **原因3**: 复杂场景难以修复
  - 解决方案：启用高质量修复模式

#### Q3: 处理时间过长
- **原因1**: 图像分辨率过高
  - 解决方案：适当降低图像分辨率
- **原因2**: 系统资源不足
  - 解决方案：关闭其他占用资源的程序
- **原因3**: 启用了高质量修复
  - 解决方案：对快速预览使用标准修复

## 6. 技术细节

### 6.1 LaMa模型原理

LaMa使用**傅里叶卷积**处理大掩码区域的图像修复：

1. **网络架构**: 基于U-Net，包含傅里叶卷积层
2. **感受野**: 大感受野处理大掩码区域
3. **多尺度特征**: 提取多尺度特征信息
4. **感知损失**: 使用预训练的VGG网络计算感知损失

### 6.2 系统架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端界面      │    │   Flask后端     │    │   LaMa模型      │
│   (HTML/JS)     │───▶│   (Python)      │───▶│   (PyTorch)     │
│                 │    │                 │    │                 │
├─────────────────┤    ├─────────────────┤    ├─────────────────┤
│  图像上传       │    │   API路由       │    │  模型加载       │
│  掩码绘制       │    │  图像处理       │    │  推理计算       │
│  结果预览       │    │  错误处理       │    │  精炼算法       │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### 6.3 性能优化

1. **图像预处理**: 自动调整到合适的分辨率
2. **批处理**: 支持批量图像处理
3. **缓存机制**: 复用已加载的模型
4. **异步处理**: 长时间任务使用异步处理
